2. Coder Model (Gemma 3b) generates the Appium code:

@Test
public void invalidLoginTest() {
  driver.findElement(By.id("username"))
    .sendKeys("testuser");
  driver.findElement(By.id("password"))
    .sendKeys("invalid_pass");
  driver.findElement(By.id("loginBtn")).click();
  WebElement error = driver.findElement(By.id("error_message"));
  Assert.assertEquals(error.getText(), "Invalid credentials");
}

The "Aha!" Moment: Testing Animations with AI

Pixel-perfect visual testing fails with dynamic content. AI understands concepts.

The Challenge: Animated UI

Pixel-matching will always fail!

→

The Solution: Conceptual Query

"Is there a lit, burning fire icon visible on the screen?"
↓
"Yes, an animated icon of a fire with moving flames is displayed."

The AI confirms the *concept* of the element, not the exact pixels.
